Request Body and Response Formats (Work in progress, may be changed)

(1) Phone format

{

        “number”: Number,

        “description”: String

}

(2) Person format

{

        “id”: Number,

        “email”: String,

        “firstName”: String,

        “lastName”: String

}

(3) Person with full info format (might be changed to a more sensible format)

{

        “id”: Number,

        “email”: String,

        “firstName”: String,

        “lastName”: String,

        “phones”: [phone, phone, ...] (1)

        “fullAddress”: {

                “street”: String,

                “additionalInfo”: String,

                “zipCode”: Number,

                “city”: String

                }

        “hobbies”: [hobby, hobby, ...] (4)

}

(4) Hobby format

{

        “name”: String,

        “description”: String

}

(5) Count format

{

        “count”: Number

}

(6) Cityinfo format

{

        “zipCode”: Number,

        “city”: String

}

(7) Generic message format

{

        “status”: String [“removed” | “error”]

}

Errors

All errors are represented using this format:

{ status: statusCode, msg: “Explain the problem” }

(e1): { status: 404, msg: “No person with number {number} found” }

(e2): { status: 404, msg: “No hobby with name {hobbyname} found” }

(e3): { status: 404, msg: “No city with zipcode {zipCode} found” }

(e4): { status: 400, msg: “Field ‘xxx’ is required” } (For example, no last name provided)
