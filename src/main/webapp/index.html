Request Body and Response Formats

(1) Phone format

{

        “number”: Number,

        “description”: String

}

(2) Person with full info format

{

        “id”: Number,

        “email”: String,

        “firstName”: String,

        “lastName”: String,

        “phones”: [phone, phone, ...] (1)

        “fullAddress”: {

                “street”: String,

                “additionalInfo”: String,

                “zipCode”: Number,

                “city”: String

                }

        “hobbies”: [hobby, hobby, ...] (3)

}

(3) Hobby format

{

        “id”: Number,

        “name”: String,

        “wikiLink”: String,

        “category”: String,

        “type”: String

}

(4) Count format

{

        “count”: Number

}

(5) Cityinfo format

{

        “zipCode”: Number,

        “city”: String

}

(6) Generic message format

{

        “status”: String [“removed” | “error”]

}

Errors

All errors are represented using this format:

{ status: statusCode, msg: “Explain the problem” }

(e1): { status: 404, msg: “No person with number {number} found” }

(e2): { status: 404, msg: “No hobby with name {hobbyname} found” }

(e3): { status: 404, msg: “No city with zipcode {zipCode} found” }

(e4): { status: 400, msg: “Field ‘xxx’ is required” } (For example, no last name provided)
